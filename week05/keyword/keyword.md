- 환경 변수
    - **정의:** 프로세스가 컴퓨터에서 동작하는 방식에 영향을 미치는 동적인 값들의 모임이다.
    - **속성 및 장점:** 소스 코드 내에 데이터베이스 비밀번호, 포트 번호, API 키 등을 하드코딩하지 않고 외부에서 설정값을 주입할 수 있어 보안성(민감 정보 보호)과 이식성(환경별 설정 변경 용이)이 뛰어나다. (예: `.env` 파일)
- CORS
    - **정의:** 웹 브라우저가 한 출처에서 실행 중인 웹 애플리케이션이 다른 출처의 선택한 자원에 접근할 수 있는 권한을 부여하도록 하는 보안 메커니즘이다.
    - **속성 및 장점:** 출처가 다른 도메인 간의 API 통신 시 발생하며, 서버에서 `Access-Control-Allow-Origin` 응답 헤더를 통해 허용된 출처만 리소스에 접근하도록 제어하여 **CSRF와 같은 보안 취약점을 방지한다**.
- DB Connection, DB Connection Pool
    - **DB Connection:** 애플리케이션과 데이터베이스 간의 물리적인 연결 세션을 의미한다. 매번 연결을 맺고 끊는 과정은 매우 무겁고 시간이 오래 걸리는 작업이다.
    - **DB Connection Pool:** 미리 일정 수의 Connection을 생성해 풀(Pool)에 담아두고, 요청이 올 때마다 빌려주고 작업이 끝나면 다시 반납받는 방식이다. **서버의 부하를 획기적으로 줄이고, 응답 속도를 크게 개선한다.**
- 비동기 (async, await)
    - **정의:** 작업이 완료될 때까지 기다리지 않고 다음 코드를 실행하는 비동기 처리를, 마치 동기식 코드처럼 직관적이고 순차적으로 작성하게 해주는 문법이다
    - **속성 및 장점:** 기존의 복잡한 콜백(Callback) 지옥이나 `Promise` 체이닝 코드를 깔끔하게 만들어 주어 **가독성을 높이고 디버깅을 용이**하게 한다.
- try/catch/finally
    - **정의:** 예외 처리(Exception Handling)를 위한 제어문이다.
    - **속성 및 장점:** - `try`: 에러가 발생할 가능성이 있는 코드를 실행한다.
        - `catch`: 에러 발생 시 애플리케이션이 멈추지 않고 예외를 잡아 처리한다.
        - `finally`: 에러 발생 여부와 상관없이 무조건 실행되는 블록으로, 데이터베이스 연결 해제나 파일 닫기 같은 **정리(Cleanup) 작업에 적합하다.**
- Interface(인터페이스)
    - **정의:** 객체의 구조나 데이터의 규격을 정의하는 TypeScript의 설계도이다.
    - **속성 및 장점:** 객체가 가져야 할 필수 속성과 선택적 속성의 형태를 명확히 하여, **컴파일 타임에 타입 안정성을 확보하고 유지보수성**을 극대화한다.
- Type Assertion(as 키워드)
    - **정의:** 개발자가 TypeScript 컴파일러보다 해당 변수의 타입을 더 잘 알고 있을 때, 특정 타입으로 강제 변환(단언)하는 기능이다.
    - **속성 및 주의사항:** `any` 타입의 데이터를 특정 인터페이스로 단언할 때 사용하지만, 런타임 시점의 실제 데이터 구조와 다를 경우 오류가 발생할 수 있으므로 주의해서 사용해야 한다.
